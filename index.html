Stripe Error Code Reference

Structured reference of Stripe API error codes, decline codes, and handling strategies.

50 errors
Error Category Cause Fix Retryable
card_declined Card Decline The card was declined by the issuing bank. The decline_code field provides more specific detail. Check the decline_code for specifics. Do not retry without user action. Ask the customer to contact their bank or use a different card. Never share the raw decline_code with the user. No
card_declined Card Decline The card does not have enough funds to complete the purchase. Ask the customer to use a different card or add funds. Do not retry the same card. No
card_declined Card Decline The card has been reported lost by the cardholder. Do not reveal this reason to avoid tipping off fraudsters. Show generic decline message. Flag the transaction for review. No
card_declined Card Decline The card has been reported stolen by the cardholder. Do not reveal the specific reason. Flag for fraud review. Block the user session if warranted. Show generic decline message. No
card_declined Card Decline Generic decline from the issuing bank with no specific reason provided. Ask the customer to contact their bank. They may need to whitelist the merchant or authorize the transaction. Retry after bank contact. No
card_declined Card Decline The transaction type is not permitted for this card. Common with prepaid cards trying to pay for subscriptions, or corporate cards with merchant category restrictions. Ask the customer to use a different card. Debit or prepaid cards often have restrictions on recurring billing. No
card_declined Card Decline The card has exceeded its spending limit for a given time period, as configured by the issuing bank. Ask the customer to contact their bank to increase limits or use a different card. Implement retry backoff if retrying failed subscription charges. No
expired_card Card Error The card's expiration date is in the past. Prompt the customer to update their card details. Use Stripe's hosted payment UI (Checkout, Payment Element) which validates expiry client-side before submission. No
incorrect_cvc Card Error The CVC/CVV number provided does not match the card. Ask the customer to re-enter the CVC. After multiple incorrect attempts, the card may be blocked by the bank. No
incorrect_number Card Error The card number fails Luhn algorithm validation or does not match a known card number format. Validate card numbers client-side using Stripe.js or a Luhn check before submission to avoid unnecessary API calls. No
invalid_expiry_month Card Error The expiry month is not a valid value (1-12). Validate expiry input client-side. Stripe.js handles this automatically when using the Payment Element. No
invalid_expiry_year Card Error The expiry year is in the past or not a valid year. Validate expiry year client-side. Prefer 2-digit year input that auto-converts to the nearest valid year. No
authentication_required 3D Secure / SCA The payment requires 3D Secure (SCA) authentication and the charge was attempted without it, or the customer did not complete 3DS. Use PaymentIntents with `stripe.confirmCardPayment()` to handle 3DS automatically. Check `payment_intent.next_action` for redirect details. Never charge cards directly in SCA-required regions. Yes
invalid_api_key Authentication The API key used is invalid, revoked, or a test key is being used in live mode (or vice versa). Verify the API key in your Stripe Dashboard. Ensure you are using the correct key for the environment (test vs live). Never expose secret keys in client-side code. No
no_account Authentication No Stripe account was found matching the provided API key or connect account ID. Verify the Stripe-Account header (for Connect) contains a valid connected account ID. Check that the account has not been deactivated. No
rate_limit Rate Limiting Too many requests made to the Stripe API in too short a time. Implement exponential backoff with jitter. Start with 1s, double each retry, up to 8 retries. Check your API usage in the Stripe Dashboard. Batch operations where possible. Yes
resource_missing Request Error The resource (customer, payment method, invoice, subscription) does not exist or does not belong to the current API key's account. Verify the resource ID is correct and exists in the right Stripe account. Test vs live mode resources are separate. Log the resource ID for debugging. No
parameter_missing Request Error A required parameter was not included in the API request. Check the Stripe API docs for required parameters. The error message will name the missing parameter. Enable Stripe request logging in development mode. No
parameter_invalid_empty Request Error A parameter was passed as an empty string where a non-empty value is required. Validate all parameters before sending to the API. Do not pass empty strings for required fields; omit them entirely or pass null. No
idempotency_key_in_use Idempotency A request is still in progress for this idempotency key. Concurrent requests with the same idempotency key are not allowed. Wait for the in-flight request to complete before retrying with the same idempotency key. Implement proper retry logic that respects in-progress requests. Yes
invalid_idempotency_key Idempotency Two requests used the same idempotency key but with different parameters — the key was not unique enough or was accidentally reused. Generate idempotency keys as UUIDs tied to the specific action (e.g. order ID + attempt number). Never reuse keys across different operations. No
subscription_payment_intent_requires_action Subscriptions / SCA An automatic subscription renewal requires 3DS authentication. This happens for SCA-mandated regions (EEA, UK) when the renewal cannot use off-session exemptions. Use `payment_intent.client_secret` from the invoice and redirect the customer to complete 3DS. Implement the subscription incomplete webhook flow with `invoice.payment_action_required` event. Yes
invoice_upcoming_none Invoices Attempting to retrieve an upcoming invoice for a customer with no active subscription. Check whether the customer has an active subscription before calling `stripe.invoices.retrieveUpcoming()`. Handle the 404 as an expected case. No
customer_tax_location_invalid Tax Stripe Tax cannot determine the customer's tax location because the address is incomplete, invalid, or conflicts between the shipping and billing address. Collect complete customer addresses (line1, city, postal_code, country). Validate addresses before sending to Stripe. Check Stripe Tax configuration in the Dashboard. No
charge_already_captured Charges Attempting to capture a charge that has already been captured. Check charge status before attempting capture. Use `charge.captured` to verify state. Implement idempotency to prevent double-capture in retry scenarios. No
charge_already_refunded Refunds Attempting to refund a charge that has already been fully refunded. Check `charge.refunded` and `charge.amount_refunded` before issuing a refund. Track refund state in your database to prevent duplicate refund attempts. No
charge_disputed Disputes The charge has an active dispute and certain operations (like refunding) are blocked during the dispute process. Resolve the dispute first. Submit evidence via the Stripe Dashboard or API. Once the dispute is resolved, you can proceed with other operations on the charge. No
balance_insufficient Payouts Attempting to create a payout larger than the available Stripe balance. Check balance before initiating payouts: `stripe.balance.retrieve()`. Set up automatic payouts in the Dashboard to avoid manual balance management. Account for payout timing differences. No
bank_account_declined ACH / Bank Transfers The ACH debit was declined by the bank. Reasons include closed account, insufficient funds, unauthorized debit, or frozen account. Contact the customer to verify bank account details. Use Stripe's bank account verification (micro-deposits or Plaid instant verification) before charging. Handle `payment_intent.payment_failed` webhooks. No
webhook_signature_verification_failed Webhooks The webhook signature cannot be verified. The raw body was modified before verification, the wrong signing secret was used, or the timestamp is outside the tolerance window. Use the raw request body for signature verification, not a parsed JSON body. Use `stripe.webhooks.constructEvent(rawBody, sig, secret)`. Verify you are using the signing secret for the correct endpoint (test vs live, different endpoints have different secrets). No
connect_account_invalid Connect The connected account ID in the Stripe-Account header is invalid, deactivated, or does not belong to your platform. Verify the connected account exists and is active: `stripe.accounts.retrieve(accountId)`. Handle account deactivation by blocking payouts to inactive accounts. No
account_country_invalid_address Connect The address provided for a Connect account does not match the country of the account. Ensure the address country matches the account's country. Validate address fields for the specific country's format (postal code format varies by country). No
platform_api_key_expired Connect The platform API key has expired. This occurs with restricted keys that have expiry set. Rotate the API key in the Stripe Dashboard and update all services using it. Implement key rotation monitoring to catch expiry before it impacts users. No
payment_intent_unexpected_state Payment Intents An operation was attempted on a PaymentIntent in an incompatible state. E.g. confirming a PaymentIntent that has already succeeded or been cancelled. Check `payment_intent.status` before performing operations. States: `requires_payment_method`, `requires_confirmation`, `requires_action`, `processing`, `succeeded`, `canceled`. No
payment_intent_incompatible_payment_method Payment Intents The payment method type is not included in the PaymentIntent's `payment_method_types` list. Update the PaymentIntent to include the desired payment method type, or create a new one with the correct types. Consider using `automatic_payment_methods: { enabled: true }` to support all Stripe-enabled methods. No
coupon_expired Coupons The coupon being applied has passed its `redeem_by` date or has exceeded its `max_redemptions` count. Check coupon validity before applying: `stripe.coupons.retrieve(couponId)`. Handle the expired state gracefully in the checkout flow. No
plan_not_found Subscriptions The Price or Plan ID referenced in a subscription does not exist or has been archived. Use the Prices API (not the deprecated Plans API). Verify the price ID in the Dashboard. Never hardcode price IDs; store them in config/env vars. No
api_connection_error Connectivity Could not connect to Stripe's servers. Usually a transient network issue, DNS failure, or firewall blocking outbound connections to api.stripe.com. Implement retry with exponential backoff. Check outbound firewall rules for api.stripe.com on port 443. Monitor Stripe status at status.stripe.com. Yes
api_error Stripe Server Error An unexpected error on Stripe's end. These are rare. Retry with exponential backoff (up to 3-5 times). If the error persists, check status.stripe.com. Log the `request_id` from the response header and include it in any support requests. Yes
currency_not_supported Currency The currency specified is not supported by the payment method or the Stripe account's country. Check supported currencies for the payment method type in Stripe docs. Verify the account's settlement currency. Use `presentment_currency` to show the customer's local currency while settling in a supported currency. No
amount_too_small Charges The charge amount is below the minimum allowed for the currency (e.g. $0.50 USD, $0.30 EUR, 50 JPY). Ensure amounts meet the minimum per-currency threshold. Check minimums in Stripe docs. Consider consolidating small charges into a single larger charge. No
amount_too_large Charges The charge amount exceeds the maximum allowed for the currency. Check maximum charge limits per currency in Stripe docs. For very large transactions, split into multiple charges or use wire transfers. No
email_invalid Customer The email provided for customer creation is not a valid email format. Validate email addresses client-side before sending to the API. Use a regex or email validation library. The `email` field on Customer is optional but must be valid if provided. No
test_mode_live_card Testing A real card number was used in test mode. Stripe blocks real card numbers in test environments. Use Stripe's test card numbers (e.g. 4242 4242 4242 4242 for Visa, 4000 0025 6000 0006 for 3DS). See the full list in Stripe's testing documentation. No
payment_method_not_available Payment Methods The payment method (SEPA, iDEAL, Klarna, etc.) is not enabled for the Stripe account or not available in the customer's region. Enable the payment method in Stripe Dashboard under Settings > Payment methods. Verify the account's country supports the method. Use `payment_method_types` to restrict what is shown. No
setup_intent_unexpected_state Setup Intents An operation was attempted on a SetupIntent in an incompatible state. Check `setup_intent.status` before operations. Valid states: `requires_payment_method`, `requires_confirmation`, `requires_action`, `processing`, `succeeded`, `canceled`. No
mandate_invalid SEPA / ACH The mandate associated with a SEPA or ACH payment method has been revoked or expired. Collect a new mandate from the customer using a SetupIntent. Check mandate status via the Mandate object. Handle `mandate.updated` webhooks to track mandate state changes. No
financial_connections_account_inactive Financial Connections The bank account linked via Stripe Financial Connections has become inactive, closed, or the access token has been revoked by the user. Prompt the user to re-link their bank account. Handle `financial_connections.account.disconnected` webhooks. Check account status before initiating transfers. No
transfer_source_transaction_insufficient_funds Connect / Transfers The source transaction used for a Connect transfer does not have sufficient funds available. Verify the charge has been captured and is not in a disputed state before transferring. Account for Stripe fees when calculating the transferable amount. No
tax_id_invalid Tax The tax ID number provided does not match the expected format for the given type and country. Validate tax ID format client-side before submission. Different types (VAT, EIN, ABN) have different formats. The error message will indicate the expected format. No
